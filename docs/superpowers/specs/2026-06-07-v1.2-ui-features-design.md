# v1.2 UI Features — Design Spec

**Date:** 2026-06-07
**Status:** Approved

---

## Overview

Four UI features for ExamDump v1.2:

1. **Splash screen** — a ~5-second branded splash window (title + version) on launch.
2. **Status bar + live clock** — a persistent bottom bar showing the current date/time (bottom-right), always visible.
3. **Resizable AI panel** — drag the divider between the quiz area and the AI/browser panel to any split (no persistence).
4. **Changelog** — an in-app screen listing version history, opened from the sidebar.

These are independent and can be built/tested separately.

---

## Feature 1: Splash Screen

**Behaviour:** On launch, a frameless, always-on-top, transparent splash window (≈420×260, centered) appears immediately and shows the app title and version. The main window is created hidden and loads behind it. After ~5 seconds the splash closes and the main window is shown.

**Architecture (main process only — no renderer/IPC changes):**

- New helper `src/splash.ts`:
  ```ts
  import { BrowserWindow, app } from 'electron';

  export function createSplashWindow(): BrowserWindow {
    const splash = new BrowserWindow({
      width: 420, height: 260, frame: false, transparent: true,
      alwaysOnTop: true, center: true, resizable: false, show: true,
      skipTaskbar: true, webPreferences: {},
    });
    const html = `<!doctype html><html><head><meta charset="utf-8"><style>
      html,body{margin:0;height:100%;font-family:Segoe UI,system-ui,sans-serif;
        background:#161b27;color:#e0e0e0;border-radius:12px;overflow:hidden;
        display:flex;flex-direction:column;align-items:center;justify-content:center;}
      .title{font-size:34px;font-weight:800;letter-spacing:1px;color:#2196f3;}
      .rule{width:120px;height:2px;background:#2d3a52;margin:14px 0;}
      .ver{font-size:13px;color:#8b9cb0;}
      .load{font-size:12px;color:#4a5568;margin-top:18px;}
    </style></head><body>
      <div class="title">ExamDump</div><div class="rule"></div>
      <div class="ver">v${app.getVersion()}</div>
      <div class="load">Loading your question banks…</div>
    </body></html>`;
    splash.loadURL('data:text/html;charset=utf-8,' + encodeURIComponent(html));
    return splash;
  }
  ```

- In `src/main.ts`, the `app.on('ready')` handler:
  1. `initDb(...)` and `registerIpcHandlers()` as today.
  2. `const splash = createSplashWindow();`
  3. `createWindow()` — but the main `BrowserWindow` is created with `show: false`.
  4. `setTimeout(() => { if (!splash.isDestroyed()) splash.close(); mainWindow?.show(); }, 5000);`

- `createWindow()` adds `show: false` to the existing `BrowserWindow` options. Everything else (preload, fuses, load URL/file) is unchanged.
- `electron-squirrel-startup` early-quit at the top of `main.ts` is untouched.

**Constant:** `SPLASH_MS = 5000` in `main.ts`.

## Feature 2: Status Bar + Live Clock

**Behaviour:** A full-width bar (~24px) along the bottom of the app, present on every screen, with the live date/time right-aligned in the format `Sat, Jun 7, 2026 · 5:42:18 PM`, updating every second. It stays visible even when the AI panel is open.

**Architecture:**

- New component `src/components/StatusBar.tsx`:
  ```tsx
  import React, { useState, useEffect } from 'react';

  export function formatClock(d: Date): string {
    const date = d.toLocaleDateString('en-US', { weekday: 'short', month: 'short', day: 'numeric', year: 'numeric' });
    const time = d.toLocaleTimeString('en-US', { hour: 'numeric', minute: '2-digit', second: '2-digit', hour12: true });
    return `${date} · ${time}`;
  }

  export function StatusBar() {
    const [now, setNow] = useState(() => new Date());
    useEffect(() => {
      const id = setInterval(() => setNow(new Date()), 1000);
      return () => clearInterval(id);
    }, []);
    return (
      <div className="status-bar">
        <span className="status-clock">{formatClock(now)}</span>
      </div>
    );
  }
  ```
  (Initial state uses a lazy initializer so there is no render-before-tick flash.)

- `src/App.tsx`: render `<StatusBar />` as the last child of `.app-shell`, after `<main>` and the panel toolbar.

- CSS in `src/styles/app.css`: `.status-bar` is `position: fixed; left:0; right:0; bottom:0; height:24px;` with a top border, dark background, flex justify-end, padding-right ~12px; `.status-clock` is `font-size:11px; color:#8b9cb0; font-family: monospace`. The `.app-shell`/`.main-panel` get `padding-bottom: 24px` (or the main panel's height accounts for the bar) so content never hides under it.

- **Native panel coupling** — in `src/browser/panel.ts`: add `const STATUS_BAR_HEIGHT = 24;` and change `sizePanelToWindow` so the panel height is `h - TOOLBAR_HEIGHT - STATUS_BAR_HEIGHT` (currently `h - TOOLBAR_HEIGHT`). This leaves the bottom 24px uncovered so the clock stays visible with the panel open.

## Feature 3: Drag-to-Resize the AI Panel

**Behaviour:** While the AI/browser panel is open, a vertical drag handle sits on the divider. Dragging it left/right changes the split between 25% and 75% panel width, live. No persistence — each time the panel opens it starts at the default 0.6 (60%).

**Architecture:**

- New pure module `src/browser/ratio.ts` (no Electron import, so it is unit-testable under Vitest):
  ```ts
  export const DEFAULT_RATIO = 0.6;
  export const MIN_RATIO = 0.25;
  export const MAX_RATIO = 0.75;

  export function clampRatio(ratio: number): number {
    return Math.min(MAX_RATIO, Math.max(MIN_RATIO, ratio));
  }
  ```

- `src/browser/panel.ts`:
  - `import { DEFAULT_RATIO, clampRatio } from './ratio';`
  - Module-level `let panelRatio = DEFAULT_RATIO;`.
  - `sizePanelToWindow` uses `const panelW = Math.floor(w * panelRatio);`.
  - New export:
    ```ts
    export function setPanelRatio(mainWindow: BrowserWindow, ratio: number): void {
      panelRatio = clampRatio(ratio);
      sizePanelToWindow(mainWindow);
    }
    ```
  - In `closePanel`, reset `panelRatio = DEFAULT_RATIO;` (imported from `./ratio`).

- IPC — the channel `RESIZE_PANEL: 'resize-panel'` already exists in `src/ipc/channels.ts`.
  - `src/main.ts`: `ipcMain.handle(IPC.RESIZE_PANEL, (_e, ratio: number) => { if (mainWindow && Number.isFinite(ratio)) setPanelRatio(mainWindow, ratio); });`
  - `src/preload.ts`: `resizePanel: (ratio) => ipcRenderer.invoke(IPC.RESIZE_PANEL, ratio),`
  - `src/types.ts` `ElectronAPI`: `resizePanel: (ratio: number) => Promise<void>;`

- Renderer — in `src/App.tsx`, when `panelOpen` is true, render a drag handle. It is `position: fixed`, full height (below the toolbar, above the status bar), `width: 6px`, `cursor: col-resize`, positioned at `left: calc(100% * (1 - ratioGuess))` — but since the renderer doesn't track the exact ratio, the handle is positioned at the panel's left edge by mirroring the same default and updating a local `panelRatio` state as the user drags. Implementation:
  - Local state `const [dragRatio, setDragRatio] = useState(0.6)` (reset to 0.6 whenever `panelOpen` transitions to true).
  - Handle style `left: calc(${(1 - dragRatio) * 100}% - 3px)`.
  - `onMouseDown` sets a `dragging` ref and adds `mousemove`/`mouseup` window listeners.
  - `mousemove`: `const ratio = (window.innerWidth - e.clientX) / window.innerWidth;` clamp 0.25–0.75; `setDragRatio(ratio)`; call `window.electronAPI.resizePanel(ratio)` (the native view repositions). Wrap the IPC call in `requestAnimationFrame` throttling (skip if a frame is already pending).
  - `mouseup`: clear `dragging`, remove listeners.
  - Reset `dragRatio` to 0.6 when the panel re-opens so the handle starts at the default split.

- CSS: `.panel-divider` for the handle (subtle, highlights on hover/active).

## Feature 4: Changelog

**Behaviour:** A 📝 Changelog button in the sidebar opens an in-app screen listing versions (newest first), each with a date heading and bulleted changes, plus a Back button.

**Architecture:**

- New data module `src/data/changelog.ts`:
  ```ts
  export interface ChangelogEntry { version: string; date: string; changes: string[]; }

  export const CHANGELOG: ChangelogEntry[] = [
    { version: '1.2.0', date: 'June 2026', changes: [
        'Splash screen on launch',
        'Live clock in the bottom status bar',
        'Drag-to-resize the AI / browser panel',
        'This changelog',
    ]},
    { version: '1.1.0', date: 'June 2026', changes: [
        'Resume an unfinished quiz where you left off',
        'Waterfall mode advances only after you finish the day’s set',
    ]},
    { version: '1.0.0', date: 'June 2026', changes: ['First release'] },
  ];
  ```

- New component `src/components/Changelog.tsx` — props `{ onBack: () => void }`. Renders a heading "What's New", maps `CHANGELOG` to version blocks (`vX.Y.Z — Date` + `<ul>` of changes), and a Back button calling `onBack`.

- `src/App.tsx`:
  - Add `{ screen: 'changelog' }` to the `AppView` union.
  - Render `<Changelog onBack={() => setView({ screen: 'library', selectedBankId: null })} />` when `view.screen === 'changelog'`.
  - Pass an `onChangelog` callback to `Sidebar`.

- `src/components/Sidebar.tsx`: add prop `onChangelog: () => void` and a nav button `📝 Changelog` calling it (in the `sidebar-nav` block).

## Testing

The renderer/Electron-window pieces (splash window, native panel sizing, drag wiring) are verified by running the app, not unit tests, consistent with the existing codebase (no React component tests; only pure logic is unit-tested).

Two small **pure helpers** are extracted and unit-tested:
- `clampRatio(ratio)` exported from `src/browser/ratio.ts` (a pure module with no Electron import) → clamps to [0.25, 0.75]. Tests: 0.6→0.6, 0.1→0.25, 0.9→0.75. (Test file: `src/browser/__tests__/ratio.test.ts`.)
- `formatClock(date)` exported from `src/components/StatusBar.tsx` → tested against a fixed `Date` for the `Weekday, Mon D, YYYY · h:mm:ss AM/PM` shape (assert it contains the date parts and a colonized AM/PM time). (Test file: `src/components/__tests__/statusBar.test.ts`.)

Manual smoke (run the app):
- Splash shows ~5s with correct version, then main window appears.
- Clock ticks every second, bottom-right, visible with the AI panel open.
- Dragging the divider resizes the AI panel between 25%–75%; reopening resets to 60%.
- Sidebar 📝 Changelog opens the list; Back returns to library.

---

## Out of Scope

- Persisting the panel split across sessions (resets to 60% by design).
- A skippable/click-to-dismiss splash (fixed ~5s).
- Markdown/file-based changelog (typed in-code by design).
- Resizing the panel vertically or detaching it into a separate OS window.
